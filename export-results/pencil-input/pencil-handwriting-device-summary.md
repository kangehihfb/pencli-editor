# 손필기 펜슬 입력 실기기 호환성 테스트 결과

생성일: 2026-04-30
테스트 문구: 안녕하세요
테스트 주소: `http://10.1.100.73:5177/?pencilReport=1`

## 목적

애플 펜슬, 와콤, S펜 환경에서 실제 스타일러스 입력으로 동일한 문구를 작성하고, 브라우저 입력 이벤트와 획 생성 결과를 기준으로 손필기 기능의 호환성을 확인한다.

## 결론

현재 구현은 아이패드 + 애플 펜슬, 갤럭시 탭 + S펜 환경에서 실기기 펜 입력을 정상 처리한다. 두 모바일 태블릿 환경 모두 포인터 유형이 펜으로 인식되었고, 필압 값이 실제로 변화했으며, 획 생성도 정상적으로 증가했다.

윈도우 PC + 와콤 환경은 필기 자체는 가능하지만 브라우저 이벤트가 펜이 아니라 마우스로 들어왔다. 따라서 획 생성 관점에서는 사용 가능하지만, 펜 전용 입력/필압/기울기 기반 품질 검증 관점에서는 주의로 분류한다.

실사용 체감 기준으로는 갤럭시 탭 + S펜의 필기감과 반응 속도가 가장 좋았다. 아이패드 + 애플 펜슬도 정상적으로 필기 가능했으며, 윈도우 PC + 와콤도 필기 기능 자체는 동작했다. 다만 와콤은 프로젝트 내부 캔버스뿐 아니라 일반 배경/화면 터치에서도 좌표가 어긋나는 양상이 관찰되어, 해당 좌표 문제는 프로젝트 단독 이슈라기보다 와콤 장비/드라이버/운영체제 입력 매핑 환경의 영향으로 보는 것이 타당하다.

실사용 관찰에서는 캔버스 내부뿐 아니라 일반 배경/화면 터치에서도 좌표점이 잘 맞지 않는 현상이 있었다. 따라서 와콤 좌표 어긋남은 현재 프로젝트의 WebGL/R3F 캔버스 처리만의 문제로 단정하기 어렵고, 와콤 드라이버, 윈도우 입력 설정, 디스플레이 배율, 브라우저 입력 매핑 조합의 영향을 함께 확인해야 한다.

## 공통 관찰

1. 아이패드와 갤럭시 탭은 실기기 펜 입력이 포인터 유형 펜으로 들어온다.
2. 와콤은 현재 테스트 환경에서 포인터 유형 마우스로 들어오므로 펜 전용 기능 검증에 한계가 있다.
3. 와콤의 좌표 어긋남은 프로젝트 외부 화면에서도 관찰되어 프로젝트 단독 이슈로 보기 어렵다.
4. 모바일 태블릿은 포인터 이벤트와 터치 이벤트가 같이 들어오는 양상이 있다.
5. 입력 캡처 레이어를 통해 실제 획 생성은 세 기기 모두 성공했다.
